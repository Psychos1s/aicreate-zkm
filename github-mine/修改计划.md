# Solution-Generator 修改计划

4. ✅ 使用业务语言描述，避免"对象"、"输入"、"输出"等技术术语
5. ✅ 阶段3不应该生成铁律
6. ✅ 添加方案检查环节

## 修改内容

### 1. 主 skill (solution-generator/SKILL.md)

**修改点1：隐藏技术细节**
- 启动流程中，静默调用子 skill，不展示 "The xxx skill is running"
- 向用户说明流程时，使用业务语言，不提及 skill、Claude Code 等

**修改点2：添加方案检查**
- 在阶段3完成后，总结前添加方案检查环节
- 检查项：
  1. 文档完整性（3个文档都存在）
  2. 约束模型完整性（如果有约束模型）
  3. 对象实例完整性（如果是特定行业领域）
- 展示检查结果，未通过给出修改建议

### 2. ability-model skill

**修改点1：基础领域也分析业务逻辑对象**
- 移除"仅当领域类型为特定行业领域时"的限制
- 所有领域都需要分析业务逻辑对象

**修改点2：使用业务语言描述**
- 向用户展示分析结果时，使用业务语言
- 避免使用"对象"、"输入对象"、"输出对象"等技术术语
- 改用"业务实体"、"依赖的信息"、"产生的结果"等业务语言

### 3. execution-model skill

**修改点：移除生成铁律**
- 删除步骤中关于生成铁律的内容
- 应用模型文档结构中移除"铁律"章节
- 只生成功能描述、流程步骤、规则约束
