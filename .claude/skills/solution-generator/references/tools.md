# 可用工具详细说明

## 1. 获取约束元工具

**功能**：调用 MCP 工具 `graph_info_query` 获取本领域的约束元（领域对象型），作为数据对象分析的参考基础。

**执行方法**：
1. 检查环境变量或 `.app_config.json` 中是否存在 `abilityModelId`
2. 若存在 `abilityModelId`：
   - 通过mcp__graph_info_query__graph_info_query获取约束模型，入参 `paperId` 取 `abilityModelId` 的值，读取约束元 JSON
   - 将 JSON 转换为模型可理解的结构化约束元格式（领域对象型：对象定义、属性表、动态属性规则、关系定义）
3. 若不存在 `abilityModelId`：表示未获取到约束元

**约束元处理**：
- **若获取到约束元**：
  - 将约束元的内容，通过清晰的格式展示给用户
  - 使用话术："已查到本领域必须涉及的核心的数据对象如下所示，请确认是否需要补充？"
  - 等待用户反馈，并根据用户反馈，更新约束元并让用户确认，直到用户明确表示不用修改为止
  - 用户仅可在此基础上补充自定义对象/规则，不可修改外部约束元内容
- **若未获取到约束元**：
  - 你将主动去分析实现用户的需求目标涉及到哪些重要的数据对象

**使用场景**：在步骤2（生成数据模型）中，首先尝试使用此工具获取约束元，作为数据对象分析的参考基础。

## 2. 领域背景知识检索工具

**功能**：基于 `knowledge-retrieval` skill 进行联网搜索，获取目标领域的背景知识，辅助需求理解。

**执行方法**：
1. 基于用户提到的行业/领域/业务关键词，构造针对性的检索 query
2. 执行检索脚本：
   ```bash
   python .claude/skills/knowledge-retrieval/scripts/retrieve_knowledge.py \
     --query "检索主题" \
     --output "report.md"
   ```
3. 读取检索报告，提取关键领域背景知识
4. 将检索结果作为需求分析的参考，不直接作为结论

**使用场景**：在步骤1（挖掘用户需求）中，当用户未上传文档且对目标领域背景不熟悉时调用。

## 3. 文档解析工具

**功能**：当用户上传文档时，调用 MCP 工具 `parse_document_content` 解析文档内容，提取文本信息供后续分析使用。

**执行方法**：
1. 识别用户上传的文档资源
2. 调用 MCP 工具 `parse_document_content`，传入文档路径或资源标识
3. 获取解析后的文本内容
4. 将解析结果作为需求分析和数据建模的输入素材

**使用场景**：在步骤1（挖掘用户需求）或步骤2（生成数据模型）中，当用户上传文档时调用。
