# 生成运行逻辑图 Skill

## 触发条件
当用户输入包含如下关键字，执行该技能
- **生成SOP**
- **生成运行逻辑图**
- **重新生成运行逻辑图**

## 核心目标
你负责根据`/output/stage3.md`生成JSON格式的运行逻辑图，并保存到 `/output/sop.json`，再调用脚本save_sop.py保存。

## 执行过程
（🔴 逐步执行`步骤1`，`步骤2`，`步骤3`，`步骤4`，`步骤5`，`步骤6`，`步骤7`一共**7**个步骤必须强制执行，不能遗漏）
### 步骤1：使用 `ReadFile` 读取 `sop-generator-md/reference/schema.json`

### 步骤2：使用 `ReadFile` 读取 `/output/stage3.md`
  - ❌禁止校验stage3.md中内容的正确性

### 步骤3: 强制删除已经存在的`/output/sop.json`文件

### 步骤4：分析`stage3.md`按照`schema.json`格式生成运行逻辑图sop.json，**如已存在则覆盖**

#### 4.1 运行逻辑图数据结构规范

##### 节点类型
- `start` - 开始节点（固定 N0）
- `task` - 任务节点
- `gateway` - 条件网关节点
- `end` - 结束节点（固定 N99）

##### ID生成规则
- N0（开始）、N99（结束）
- 中间节点：N1, N2, N3...
- 继续编辑时检查避免冲突

##### 完整节点示例
```json
{
  "id": "N1",
  "type": "task",
  "name": "节点名称",
  "desc": "任务描述",
  "input": {"desc": "输入描述"},
  "output": {"desc": "输出描述"},
  "businessLogics": "执行步骤",
  "completeCondition": {"businessRule": "完成规则"}, 
  "gatewayRules": [{
    "ruleText": "条件分支1",
    "targetNodeId": "目标节点id"
  }, {
    "ruleText": "条件分支2",
    "targetNodeId": "目标节点id"
  }]
}
```
#### 4.2 运行逻辑图生成规则
按照如下原则提取`stage3.md`，按照`schema.json`格式生成节点和边并输出到`/output/sop.json`。
❌ **（不要调用任何MCP工具）**
##### 1. 生成开始节点 `N0`，**message字段由AI根据应用名称和描述动态生成欢迎语**：
   ```json
   {"id": "N0", "type": "start", "name": "开始", "message": "AI生成的欢迎语"}
   ```
##### 2. 生成中间节点（任务 & 网关）及边

数据来源严格基于 stage3.md，遵循以下 Schema：
- 解析stage3.md文档，判断是否包含标题`交互逻辑`关键字：
  - 如果章节标题不包含`交互逻辑`关键字，通过AI进行分析生成任务节点、边。
    - 任务节点：提取文档中带有明显`业务过程`,`步骤`、`阶段`、`顺序`性质的标题生成任务节点
    - 边生成（**必须严格遵守如下规则**）： 
      - 从`NO`开始到`N99`结束，依次为节点添加边
      - ❌所有的边都**同一方向**，**不能出现循环的边**，确保所有边**单向无环**
      - ❌同一任务节点不能出现两条边指向下一个节点
      - ❌不要解析章节里面的依赖关系，统一按照顺序生成边
  - 如果章节标题包含`交互逻辑`关键字，根据交互逻辑下的子标题生成任务节点、边生成
    - 任务节点生成
      - 分析标题`交互逻辑`下的子标题，按照**子标题**顺序生成任务节点
      - 节点`type`为`task`，根据`数据依赖`和`交付物`装填`input`和`output`
      - 节点`name`装填子标题名称
      - 节点`desc`装填`业务描述`
      - 从`业务描述`、`注意事项`分析提取节点的完成条件，并写入到`completeCondition`完成规则
      - 从`业务描述`、`注意事项`分析提取节点的执行步骤，并写入到`businessLogics`执行步骤
    - 边生成（**必须严格遵守如下规则**）： 
      - 从`NO`开始到`N99`结束，依次为节点添加边
      - ❌所有的边都**同一方向**，**不能出现循环的边**，确保所有边**单向无环**
      - ❌同一任务节点不能出现两条边指向下一个节点
      - ❌不要解析章节里面的依赖关系，统一按照顺序生成边
##### 3. 生成结束节点 `N99`，**message字段由AI根据应用名称生成结束语**：
   ```json
   {"id": "N99", "type": "end", "name": "结束", "message": "AI生成的结束语"}
   ```
### 步骤5 校验sop.json是否符合如下规范，如果不符合规范就重新生成（静默执行）
- ❌所有的边都**同一方向**，**不能出现循环的边** 
- ❌同一任务节点不能出现两条边指向下一个节点
- 所有边同一方向（N0 → N99），确保所有边**单向无环**
- 必须包含`N0`开始节点和`N99`结束节点

### 步骤6 使用 Bash 工具执行以下命令
`python .claude/skills/sop-generator-md/scripts/save_sop.py`
- 脚本的路径为 `.claude/skills/sop-generator-md/scripts/save_sop.py`，不要自作主张去修改。
- 如果路径下找不到`save_sop.py`这个脚本，就全局搜索下`save_sop.py`，然后再执行。

### 步骤7 识别skill之间的调用关系，判断本技能是否是由父技能调用
  - 如果是父技能调用：
    - 输出：`运行逻辑图已生成，继续执行`，同时返回到**父技能**继续执行剩余的步骤
  - 如果不是父技能调用：
    - 输出：`运行逻辑图已生成`
