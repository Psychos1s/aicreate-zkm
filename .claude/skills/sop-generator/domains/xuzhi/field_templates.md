# 需智领域字段生成模板

## 名称模板

**格式：** 动词 + 名词，6-12字

**示例：**
- ✅ 初步信息收集、生成客户画像、推导模糊意图
- ❌ 信息处理（禁用：处理/管理/操作/执行/进行）

**推荐动词：** 收集、生成、推导、确认、构建、分析、识别

---

## 描述模板

**格式：** 基于【输入】，通过【方法】，生成【输出】，支持【交互】，最终【目标】

**示例：**
```
根据客户行业、基本信息及相似企业数据，自动生成初步信息收集问卷，
支持三种收集方式（Word下载/在线填写/对话补充）；
对生产型企业自动触发深度知识挖掘Agent。
```

---

## 业务逻辑模板

```
1.**步骤名**：输入→处理→输出

2.**判断**：
  - 如果 【条件A】：执行 【动作A】 → 输出 【结果A】
  - 否则如果 【条件B】：执行 【动作B】 → 输出 【结果B】
  - 否则：执行 【默认】 → 输出 【默认结果】

3.**引导规则**：向用户说明"我为你做了什么"+"下一步能做什么"
```

**生产型企业判断：**
- 关键词：生产、制造、加工、装配、产线、工艺、设备
- 行业：制造业、能源、化工、食品加工、汽车制造

---

## 输入模板

```
【来源】：来自节点{X}/系统/用户
【对象】：{数据对象名称}
【关键字段】：
- {字段1}：{说明}
- {字段2}：{说明}
```

---

## 输出模板

```
【名称】：{产出物名称}
【结构】：多维度数据/有序列表/结构化表单/在线文档/摘要文本
【关键内容】：
- {内容1}：{说明}
- {内容2}：{说明}
【使用方式】：{下游如何使用}
```

---

## 完成标准模板

```
1.【产出确认】{具体产出物}已{状态：生成/确认/保存}
2.【用户确认】用户已{动作：查看确认/补充调整/提交}
3.【分支处理】{特殊条件}已处理（如适用）
```

---

## 质量检查清单

| 字段 | 检查点 |
|------|--------|
| 名称 | 6-12字、动词+名词、无禁用词 |
| 描述 | 50-250字、5要素齐全 |
| 业务逻辑 | 3+步骤、判断分支、引导规则 |
| 输入输出 | 标明来源/结构、列出字段 |
| 完成标准 | 产出确认、用户确认 |

---

## 常见错误修正

**错误1：业务逻辑笼统**
- ❌ 1.收集 2.分析 3.输出
- ✅ 1.**问卷生成**：基于输入生成问卷→输出问卷 2.**方式选择**：如果选Word→下载模板...

**错误2：输入输出模糊**
- ❌ 输入：客户信息 / 输出：分析结果
- ✅ 【来源】系统数据【对象】客户信息【字段】行业、规模、战略目标

**错误3：完成标准不清**
- ❌ 用户确认即可
- ✅ 1.【产出】问卷已生成 2.【用户】已确认 3.【分支】挖掘已处理（如适用）
